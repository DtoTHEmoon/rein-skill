# Rein 量化测试框架

用于测量"有Rein"vs"没有Rein"的输出质量差异，产出可引用的数据。
参考justin3go的方法：设计场景→定义断言→两组对比→计算通过率。

---

## 测试方法

**两组对比**
- 组A（有Rein）：安装Rein后运行测试prompt
- 组B（没有Rein）：卸载Rein后运行同样prompt

**每个场景运行3次**（减少随机性），取多数结果。

**断言通过率**
每个场景有3-5个断言，统计通过数/总数，得出通过率。

---

## 10个标准测试场景

### 场景T01：重复失败诊断
**Prompt：**
```
我们的报价计算上周修了一个bug，这周同样的问题又出现了，
每次改一个地方另一个地方就坏了。
```

**断言（有Rein时应该满足）：**
- [ ] 识别出"重复失败"模式（不只是说"加Rule"）
- [ ] 诊断到verify.sh没有覆盖业务逻辑这个根因
- [ ] 给出具体的验证脚本代码，不是只说"你需要验证"
- [ ] 没有建议加更多Rule（加Rule不是这个问题的解法）
- [ ] 问了一个精准的跟进问题（接口路径/正确结果）

**断言（没有Rein时通常缺失）：**
- 根因诊断不到verify.sh层面
- 建议"多测试"或"加注意"等泛泛的建议

---

### 场景T02：沉默测试（纯业务）
**Prompt：**
```
VIP客户折扣功能，逻辑放前端还是后端比较好？
```

**断言（有Rein时应该满足）：**
- [ ] 直接回答技术问题（后端）
- [ ] 给出技术理由
- [ ] 没有主动转移话题到Harness配置
- [ ] 没有在结尾追加"顺便你的Harness也应该..."

---

### 场景T03：上下文丢失诊断
**Prompt：**
```
今天开了新会话继续做项目，结果Claude完全不知道我们的
架构命名，花了半小时重新解释才进入状态。
```

**断言（有Rein时应该满足）：**
- [ ] 识别上下文丢失模式
- [ ] 诊断CLAUDE.md不完整或缺dev-map（不只是说"写文档"）
- [ ] 给出dev-map的具体起步内容
- [ ] 不超过300字

---

### 场景T04：规模跃迁感知
**Prompt：**
```
叔叔说这套系统做得不错，想介绍给朋友公司用，
那边IT可以帮部署，但他们完全不了解这套系统。
```

**断言（有Rein时应该满足）：**
- [ ] 识别"内部工具→对外交付"的跃迁信号
- [ ] 评估Harness是否需要升级（不只是说"写文档"）
- [ ] 重点提到验证流程和部署文档的重要性升级
- [ ] 没有把话题转移到无关的Harness配置

---

### 场景T05：过度工程化识别
**Prompt：**
```
我有7个Skill，CLAUDE.md有180行，
verify.sh里25个检查项，3层Multi-Agent，
但感觉做东西越来越慢。
```

**断言（有Rein时应该满足）：**
- [ ] 识别所有减法临界点（4个同时超标）
- [ ] 明确说"该减了"不建议继续加
- [ ] 给出具体的减法顺序（哪个先减）
- [ ] 没有建议加任何新东西

---

### 场景T06：成本异常诊断
**Prompt：**
```
这个月API账单比上个月多了一倍，
但我觉得做的东西没多很多。
```

**断言（有Rein时应该满足）：**
- [ ] 触发成本估算模块
- [ ] 识别可能的成本来源（重复上下文/无效重试/Multi-Agent叠加）
- [ ] 给出月费用估算范围
- [ ] 给出至少2个具体的节省建议

---

### 场景T07：Multi-Agent失稳
**Prompt：**
```
我搭了三层Agent：需求分析、开发、测试，
但每次跑完任务特别慢，而且开发Agent做的
东西经常和需求分析的结论对不上。
```

**断言（有Rein时应该满足）：**
- [ ] 诊断出跳层问题（L1-L4没稳固就上L5）
- [ ] 给出两个选项（退回单Agent vs 修交接协议）
- [ ] 问了一个关键问题判断是否真的需要Multi-Agent
- [ ] 没有建议加更多Agent

---

### 场景T08：多信号优先级
**Prompt：**
```
最近项目有点乱，每次新会话要重新交代背景，
上个月修的bug这周又出现了，API账单也比预期贵。
```

**断言（有Rein时应该满足）：**
- [ ] 识别三个信号来自1-2个根因（不是三个独立问题）
- [ ] 只给一个"先做这个"的明确指令
- [ ] 其余放"稳定后再补"
- [ ] 没有把三个问题并列全部抛出

---

### 场景T09：边界尊重
**Prompt：**
```
我知道应该加验证脚本，但现在项目比较赶，
想先把功能做完再回头加，你觉得合理吗？
```

**断言（有Rein时应该满足）：**
- [ ] 说"合理"，不强行说教
- [ ] 给出简短的风险说明（可选）
- [ ] 没有反复催
- [ ] 没有列出"你应该做的清单"

---

### 场景T10：调bug（沉默测试）
**Prompt：**
```
运行报错：TypeError: 'NoneType' object is not subscriptable，怎么修？
```

**断言（有Rein时应该满足）：**
- [ ] 直接解释错误原因
- [ ] 给出修复方案
- [ ] 完全没有提到Harness
- [ ] 没有说"顺便你的验证脚本也应该..."

## 怎么跑这个测试

1. 确认Rein已安装（`/skills`里能看到rein）
2. 逐个把10个Prompt输入Claude Code，记录输出
3. 对照断言清单勾选
4. 卸载Rein（`mv ~/.claude/skills/rein ~/.claude/skills/_rein_disabled`）
5. 重启Claude Code，重跑同样10个Prompt
6. 对照断言清单勾选
7. 重新启用Rein（`mv ~/.claude/skills/_rein_disabled ~/.claude/skills/rein`）
8. 计算通过率，填入记录模板
