【为什么这个缺口导致你的问题】
没有验证脚本，"改完"的判定依赖 AI 自己说——AI 不知道上周那个修复依赖的隐性约束，所以每次改都可能踩回去。

【怎么补——从这里开始】

在项目根目录创建 `verify_quote.sh`，第一步只写三个核心用例：

```bash
#!/bin/bash
FAIL=0

result=$(node -e "const q = require('./quote'); console.log(q.calc({...}))")
[ "$result" = "预期值1" ] || { echo "❌ 用例1失败: $result"; FAIL=1; }

result=$(node -e "const q = require('./quote'); console.log(q.calc({...折扣场景...}))")
[ "$result" = "预期值2" ] || { echo "❌ 用例2失败: $result"; FAIL=1; }

[ $FAIL -eq 0 ] && echo "✅ 报价验证通过" || exit 1
```

**规则**：每次让 AI 改报价逻辑，改完先跑 `./verify_quote.sh`，脚本通过才算做完。

## T06

账单翻倍但产出没变，通常是三个原因之一，先定位再行动。

**快速自查——哪个最像你的情况：**

**A. Multi-Agent 协调开销**
T05 里你有 3 层 Multi-Agent。每层之间传递的上下文会叠加计费，3 层意味着同样的内容可能被发送 3-4 次。上个月有没有新增或扩展 Multi-Agent 链路？

**B. verify.sh 触发了大量重试**
25 个检查项，如果有几项容易失败，Claude 会反复尝试修复——每次重试都是一次完整的上下文调用。检查一下日志，看单个任务的对话轮数有没有变长。

**C. CLAUDE.md 180 行每次都全量载入**
180 行 × 每天调用次数 × 每次对话轮数 = 隐性 token 消耗。这部分是静默涨价，很难察觉。

**最快见效的一刀：** 把 CLAUDE.md 从 180 行裁到 60 行以内（T05 减法第三步提前做），立竿见影减少每次调用的基础 token。

## T07

这两个症状来自同一个根因：**Agent 之间没有结构化的交接文档，上下文在传递时丢失了。**

需求分析 Agent 的结论停在它自己的会话里，开发 Agent 拿到的是摘要或截断版，自然对不上。

【怎么补——从这里开始】

在需求分析 Agent 的 prompt 末尾加一条强制输出：

```
任务结束前，必须输出一个 spec.md 文件，格式如下：

## 需求结论
- 要实现什么：[一句话]
- 边界条件：[列举，不超过5条]
- 明确不做的：[列举]

## 开发 Agent 的入参
- 改动范围：[具体文件或模块]
- 验收标准：[可量化，verify.sh 能检查的]
```

开发 Agent 的 prompt 第一行改成：**"先读 spec.md，以其中的验收标准为准，有疑问停下来问而不是自行决定。"**

这一步做完，"对不上"的问题会减少 80%。

**慢的问题**单独说：三层串行本身就慢，这是结构代价。但如果需求分析和测试之间没有依赖，可以让测试 Agent 在开发完成后立即并行跑，而不是等需求分析 Agent 完全退出。

先把交接文档做好，再考虑并行——顺序不能反。
